Arbres de Beth & déduction

C11 — Arbres de vérité, consistance, conséquence logique

1. Introduction

Les arbres de Beth (aussi appelés arbres de vérité) peuvent remplacer les tables de vérité et les tableaux de Karnaugh. Ils permettent de :

• Trouver les distributions de vérité qui rendent une formule F vraie
• Déduire des FND ou FNC souvent simplifiées par rapport aux tables de vérité
• Montrer qu'une formule est une tautologie
• Montrer la (non-)consistance d'un ensemble de formules
ℹ L'appellation « arbres de Beth » est en l'honneur du logicien néerlandais Evert Willem Beth (1908–1964).

2. Règle pour la conjonction (∧)

Si les deux formules A et B sont des conséquences de la formule initiale, on ajoute les deux sur la même branche.

   A ∧ B
     │
     A        ← même branche (non-branching)
     │
     B

Explication : si A∧B est vrai, alors A et B sont vrais simultanément.

3. Règle pour la disjonction (∨)

Si l'une au moins des formules A ou B est une conséquence de la formule initiale, on crée une branche pour A et une pour B.

   A ∨ B
    / \
   A   B      ← branchement (branching)

Explication : au moins l'un des deux est vrai, mais on ne sait pas lequel → on explore les deux cas.

4. Règle pour la négation (¬)

La négation est simplifiée en utilisant les lois de De Morgan :

¬(A ∧ B)  eq  (¬A) ∨ (¬B)     → branchement : ¬A | ¬B
¬(A ∨ B)  eq  (¬A) ∧ (¬B)     → même branche : ¬A puis ¬B

Schéma pour ¬(A ∧ B) :

  ¬(A ∧ B)
    / \
  ¬A   ¬B

Schéma pour ¬(A ∨ B) :

  ¬(A ∨ B)
      │
     ¬A
      │
     ¬B

5. Règle pour l'implication (→)

A → B      eq  (¬A) ∨ B        → branchement : ¬A | B
¬(A → B)   eq  A ∧ (¬B)        → même branche : A puis ¬B

Schéma pour A → B :

   A → B
    / \
  ¬A   B

Schéma pour ¬(A → B) :

  ¬(A → B)
      │
      A
      │
     ¬B

6. Règle pour l'équivalence (⟷)

A ⟷ B     eq  (A → B) ∧ (B → A)
            eq  (¬A ∨ B) ∧ (¬B ∨ A)
            eq  (A ∧ B) ∨ (¬A ∧ ¬B)   → branchement

Schéma :

    A ⟷ B
     /   \
    A    ¬A       ← branche gauche : A puis B
    |     |
    B    ¬B       ← branche droite : ¬A puis ¬B
¬(A ⟷ B)  eq  ¬((¬A∨B) ∧ (¬B∨A))
            eq  (A ∧ ¬B) ∨ (B ∧ ¬A)    → branchement

Schéma :

   ¬(A ⟷ B)
     /    \
    A     ¬A       ← branche gauche : A puis ¬B
    |      |
   ¬B      B       ← branche droite : ¬A puis B

8. Processus inductif

De façon inductive, on décompose de la même manière les sous-formules obtenues.

Lorsqu'une sous-formule est un atome (variable propositionnelle) ou sa négation, c'est qu'on a atteint une feuille de l'arbre : on s'arrête.

9. Exemple détaillé : arbre de F

Soit F = ((¬a ∨ b) ∧ c) → (a ∧ ¬b)

F est une implication A → B avec A = (¬a∨b)∧c et B = a∧¬b.

          ((¬a ∨ b) ∧ c) → (a ∧ ¬b)                 ← formule initiale
                   /              \
        ¬((¬a ∨ b) ∧ c)        (a ∧ ¬b)             ← → donne branchement
             /      \              |
      ¬(¬a ∨ b)     ¬c            a                  ← ¬∧ = branchement ; ∧ = même branche
           |                      |
        a ∧ ¬b                   ¬b
           |
           a
           |
          ¬b

Résultat : 3 branches, mais 2 ont les mêmes atomes (a, ¬b) :

Branche 1 : a, ¬b       (issue de ¬(¬a∨b) → a∧¬b)
Branche 2 : ¬c           (issue de ¬c)
Branche 3 : a, ¬b       (issue de a∧¬b directement)
Lecture : Si l'on rend vrais tous les atomes d'une branche, on rend vraies toutes les formules de cette branche, donc la formule d'origine. Il suffit que (a∧¬b) ou ¬c soit vrai pour que F le soit.

10. Arbre de F → FND

Chaque branche ouverte correspond à un terme conjonctif (conjonction des atomes de la branche). L'ensemble des branches donne la FND.

Pour F = ((¬a∨b) ∧ c) → (a ∧ ¬b) :

Branches 1 et 3 : a ∧ ¬b    (identiques, on n'en garde qu'un)
Branche 2 :        ¬c

FND de F = (a ∧ ¬b) ∨ ¬c
✓ Cette FND est souvent simplifiée par rapport à celle obtenue par la méthode des tables de vérité.

11. Arbre de ¬F → FNC

Pour obtenir une FNC de F, on construit l'arbre de ¬F.

¬F = ¬(((¬a ∨ b) ∧ c) → (a ∧ ¬b)) = ((¬a ∨ b) ∧ c) ∧ ¬(a ∧ ¬b)

     ((¬a ∨ b) ∧ c) ∧ ¬(a ∧ ¬b)          ← ∧ même branche
                  |
          (¬a ∨ b) ∧ c                     ← ∧ même branche
                  |
              ¬a ∨ b                        
                  |
                  c
                  |
            ¬(a ∧ ¬b)                      ← ¬∧ = branchement
              /     \
            ¬a       ¬(¬b)
                       |
                       b

Deux branches ouvertes :

Branche gauche : c, ¬a    → terme conjonctif : c ∧ ¬a
Branche droite : c, b     → terme conjonctif : c ∧ b

FND de ¬F = (c ∧ ¬a) ∨ (c ∧ b)

Pour obtenir la FNC de F :

F = ¬(¬F) = ¬((c ∧ ¬a) ∨ (c ∧ b))
  = ¬(c ∧ ¬a) ∧ ¬(c ∧ b)          ← De Morgan
  = (¬c ∨ a) ∧ (¬c ∨ ¬b)

FNC de F = (¬c ∨ a) ∧ (¬c ∨ ¬b)

12. Branches fermées et arbres fermés

Branche fermée (notée ✗) : une branche qui comporte une variable propositionnelle et sa négation (p et ¬p). C'est contradictoire.

Arbre fermé : toutes les branches sont fermées → la formule d'origine n'est pas satisfaisable.

Branche ouverte : une branche complètement développée sans contradiction → donne un modèle satisfaisant.

Tautologie : pour montrer que F est une tautologie, il suffit de montrer que l'arbre de ¬F est fermé (¬F n'est pas satisfaisable → F est toujours vraie).

13. Notion de conséquence logique

Ensemble satisfaisable : un ensemble de formules ∑ est dit satisfaisable s'il existe une distribution δ qui satisfait toutes les formules de ∑ à la fois :

∃δ, ∀F ∈ ∑, δ(F) = 1

Conséquence logique : soit C une formule n'appartenant pas à ∑. La formule C est dite conséquence de ∑ si toute distribution satisfaisant toutes les formules de ∑ satisfait aussi C :

∀δ, (∀F ∈ ∑, δ(F) = 1) ⟹ δ(C) = 1

On note ∑ ⊨ C. ∑ est l'ensemble d'hypothèses, C est la conclusion.

14. Arbres de Beth et consistance d'un ensemble

Soit un ensemble de formules {H₁, H₂, …, Hₙ}. On lui associe un arbre dont ces formules sont les formules initiales : elles forment le tronc de l'arbre.

On développe l'arbre de Beth :

• S'il existe au moins une branche ouverte :
  → il existe une distribution satisfaisant toutes les formules
  → l'ensemble est dit consistant (satisfaisable)

• Si toutes les branches sont fermées :
  → les formules ne peuvent pas être simultanément satisfaites
  → l'ensemble est dit inconsistant (insatisfaisable)

15. Arbres de Beth et légitimité d'une déduction

Peut-on déduire la formule C à partir de l'ensemble de formules ∑ ?

Méthode : on teste si l'ensemble ∑ ∪ {¬C} est consistant ou non.

Tronc de l'arbre = hypothèses de ∑ + négation de la conclusion (¬C)

• Si l'arbre est fermé (inconsistant) :
  → il n'existe aucune distribution satisfaisant ∑ et ¬C
  → toute distribution qui satisfait ∑ satisfait aussi C
  → la déduction ∑ ⊨ C est LÉGITIME

• Si l'arbre a une branche ouverte (consistant) :
  → il existe une distribution satisfaisant ∑ et ¬C (contre-exemple)
  → la déduction ∑ ⊨ C n'est PAS légitime

16. Exemples de déduction

Exemple 1 : (p ∧ q) permet de déduire p ?

Tronc : p ∧ q , ¬p

   p ∧ q       ← ∧ même branche
     |
     p
     |
     q
     |
    ¬p          ← contradiction : p et ¬p → branche fermée ✗

Arbre fermé → déduction {p∧q} ⊨ p est LÉGITIME ✓

Exemple 2 : p permet de déduire (p ∨ q) ?

Tronc : p , ¬(p ∨ q)

     p
     |
  ¬(p ∨ q)     ← ¬∨ = même branche (De Morgan)
     |
    ¬p          ← contradiction : p et ¬p → branche fermée ✗
     |
    ¬q

Arbre fermé → déduction {p} ⊨ (p∨q) est LÉGITIME ✓

Exemple 3 : ∑ = {p, q → (p → r)}. Peut-on déduire C = q → r ?

Tronc : p , q → (p → r) , ¬(q → r)

¬(q → r)   eq  q ∧ ¬r         ← ¬→ même branche : q, ¬r
q → (p → r)                    ← → branchement : ¬q | (p → r)
  /           \
¬q            p → r
 |              / \
 q ← contradiction  ¬p    r
      ✗               |    |
                       p    ¬r ← contradiction
                    ← contradiction  ✗
                       ✗

Toutes les branches fermées → déduction ∑ ⊨ (q→r) est LÉGITIME ✓

Exemple 4 : ∑ = {(p∧q)→r, r→s, q∧¬s}. Peut-on déduire C = ¬p ?

Tronc : (p∧q)→r , r→s , q∧¬s , ¬(¬p) = p

q∧¬s          ← ∧ même branche : q, ¬s
(p∧q)→r       ← → branchement : ¬(p∧q) | r
   /              \
¬(p∧q)            r
  / \              |
¬p   ¬q          r→s       ← → branchement
 |    |           / \
 p    q         ¬r   s
 ✗    ✗          |    |
                 r    ¬s
                 ✗    ✗

Toutes les branches fermées → déduction ∑ ⊨ ¬p est LÉGITIME ✓

17. Remarques pratiques

✓ Conseils pour construire un arbre :
• On peut traiter les formules du tronc dans un ordre quelconque : commencer de préférence par les conjonctions (elles ne créent pas de branches).
• Dès qu'une branche est fermée, on n'a pas besoin de reproduire dans sa suite d'éventuelles sous-formules.
• Lorsqu'on traite une formule, on écrit les sous-formules à l'extrémité de toutes les branches ouvertes qui passent par la formule considérée.
• Marquer une formule traitée (✓) pour ne pas la traiter deux fois.
• Numéroter les formules et leurs enfants pour faciliter le suivi.

18. Propriétés de la déduction

Propriété 1 — Ensemble inconsistant :

⚠ Si ∑ est un ensemble de formules inconsistant, alors pour toute formule C, la déduction ∑ ⊨ C est légitime. « D'un ensemble de formules inconsistant, on peut déduire n'importe quoi. »

Exemple : l'ensemble des proverbes est inconsistant (ex. « tel père, tel fils » et « à père avare, enfant prodigue » sont des négations l'un de l'autre). On peut donc déduire de l'ensemble des proverbes n'importe quel énoncé.

Propriété 2 — Tautologie :

Si C est une tautologie, C est déductible de tout ensemble de formules ∑. En particulier, une tautologie est déductible de l'ensemble vide ∅.

Propriété 3 — Monotonie :

Si ∑ ⊂ ∑' et ∑ ⊨ C, alors ∑' ⊨ C. Si une déduction est légitime, on peut toujours ajouter des hypothèses, la déduction reste légitime.

Notation ⊨ :

• ∑ ⊨ C    →  « C est déductible de ∑ »
• ⊨ F      →  « F est tautologique »  (abréviation de ∅ ⊨ F)

Il est cohérent d'utiliser le même symbole ⊨ pour les deux notations.

19. Théorème de la déduction

Théorème : Si ∑ est un ensemble de formules et H une formule donnée, alors :

∑ ∪ {H} ⊨ C    ⟺    ∑ ⊨ (H → C)

On peut « déplacer » une hypothèse H du côté gauche vers une implication dans la conclusion.

Cas particulier ∑ = ∅ :

H ⊨ C    ⟺    ⊨ (H → C)

C'est-à-dire : C est déductible de H si et seulement si H → C est une tautologie. C'est le lien fondamental entre le connecteur → et la déduction ⊨.

Remarque : ∑ = {H₁, H₂, …, Hₙ} est un ensemble d'hypothèses. Il est équivalent d'écrire :

{H₁, H₂, …, Hₙ} ⊨ C    ⟺    H₁ ∧ H₂ ∧ … ∧ Hₙ ⊨ C
